Academic Website Starter Kit

{{ .Title }}

Gary King and Katalina Toth

Professional researchers are by definition public figures and essentially all have websites, usually beginning in graduate school. If you are either starting out your academic career and want a website like Katalina's (a post-bac at Harvard's IQSS), or have lots more material and need a more extensive website like Gary's (as a post-post-post-…bac. faculty member and director of IQSS), then this page provides the tools to build your first version in 15 minutes. You can also use our tools to improve your site as you like without any coding knowledge, friction, or annoyance. Really, it's easy. And hosting is free; the site is lightning fast; and, although it uses all the best modern tools, you don't need to know any of them.

Get your site up and running

This section is all you need. Just follow the steps — it takes about 15 minutes. Click any step to expand it.

★ Before you begin — what you'll need ▾
  • Cursor — download it free at cursor.com. During setup, create an account and pick a plan that includes Claude (Pro or above).
  • A GitHub account — sign up at github.com if you don't have one.
  • A few files (optional) — your CV PDF, a headshot, and any paper PDFs you'd like hosted.
1 Get your prompt document ▾

Download the prompt that fits where you are in your career, then fill it in:

{{ range $prompts }} {{ upper (strings.TrimPrefix "." (path.Ext .file)) }} {{ .name }}{{ .meta }} ↓ {{ end }}

You have two options:

  • Building from scratch: fill in the "Who I Am" section with your information, publications, software, etc.
  • Migrating an existing site: just paste the URL of your current website — Claude will pull the content from it.
2 Create a GitHub repository ▾

Go to github.com, click the + icon in the top right, and select New repository. Name it something like yourname-site or yourusername.github.io. Set it to Public. Don't add any files — leave it empty.

3 Open Cursor and start a chat with Claude ▾

Open Cursor. Press Cmd+I (Mac) or Ctrl+I (Windows) to open the chat.

4 Give Claude your completed prompt ▾

Attach your filled-out prompt document (plus any PDFs/photos) and say:

Say this to Claude "Create a new academic website for [Your Name], using the website building instructions attached. The GitHub repository I'd like to use is at https://github.com/YOURUSERNAME/REPONAME. Install Hugo so that we can do local builds. Build locally first and give me a localhost link so I can preview."

If migrating an existing site, say instead:

If migrating "Create a new academic website for [Your Name] by migrating content from [your existing site URL], using the website building instructions attached. My GitHub repository is at [repo URL]. Build locally first so I can preview."

Claude will scaffold the site. This may take a few minutes for large sites.

5 Preview and approve ▾

Claude will give you a link like localhost:1313. Click it to browse your site. Tell Claude what to fix — layout, missing content, ordering, anything. Iterate until you're satisfied. You can speak to Claude like you would a person who is designing your site.

Good to know This preview runs locally — only on your own computer. No one else can see it and nothing is public yet. Your site doesn't go live on the internet until you push it in the next step.
6 Push it live ▾

When ready:

Say this to Claude "Looks good. Commit and push to main."

Your site will be live within a few minutes. Using a custom domain (e.g., yourname.harvard.edu)? Tell Claude "Set up a custom domain pointing to [your domain]" and follow your university's DNS instructions.

Keep handy for later

Optional — handy once your site is live. You don't need any of this to get started.

@ Change your web address (use a custom URL) ▾

Your free site lives at an address like yourusername.github.io. You can point a nicer, custom address at it — two common options:

  • Use your university's address. Many universities will give you something like yourname.harvard.edu. Ask your department or IT group to set it up — they'll add a DNS record that points to your site.
  • Buy your own address. Purchase a domain like yourname.com from a registrar such as Namecheap, Cloudflare, or Squarespace (usually about $10–15 a year).

Either way, once you have the address, just tell Claude:

Say this to Claude "Set up a custom domain for my site pointing to [your address, e.g. yourname.harvard.edu], and tell me exactly which DNS records to add."

Claude will configure the site and hand you the exact records to paste into your university's form or your registrar's dashboard.

+ After it's live: common update prompts ▾

Open Cursor anytime to make updates. Some examples:

Add a journal article"Add a new journal article to my site: Title: [title] · Authors: [names] · Venue: [e.g., American Political Science Review, 111, 3, Pp. 484–501] · Year: [year] · Abstract: [paste] · DOI or URL: [link] · Topic: [research area]. The article PDF is attached. Build locally first."
Add a custom short URL"Add a new short URL redirect: Short path: /myproject (becomes yoursite.com/myproject) · Destination: [full URL or site path]. Push when done."
Update a person's profile (group sites)"Update [Name]'s profile: change their title to [new title] and replace their photo with the attached image. Push when done."
Add a research area"Add a new research area called '[Name]' with this description: [1–2 sentences]. Assign these existing publications to it: [list titles]. Push when done."
Add a presentation or talk"Add a new talk: Title: [title] · Event: [venue] · Date: [date] · Slides PDF attached. Push when done."

If several people edit the site: before starting, tell Claude "Pull any recent changes to main first." This prevents conflicts.

! If something goes wrong ▾
  • Site isn't updating after a push: go to your repository on GitHub, click the Actions tab. If there's a red X, copy the error and tell Claude: "The build failed. Here's the error: [paste]."
  • You don't like a recent change: tell Claude "Revert the last commit and push." Everything is versioned, so nothing is lost forever.
